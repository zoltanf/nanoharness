How Tool Calls Work in LLMs

The request → tool use → response loop

1
📨 User sends a prompt + tool definitions
👤 User

"What's the weather in Vienna today?"

⚙️ API / Orchestrator

Wraps the message with available tool schemas and sends everything to the LLM.

{ "messages": [{ "role": "user", "content": "..." }], "tools": [ { "name": "get_weather", "description": "Get current weather", "input_schema": { "location": "string" } } ] }
2
🧠 LLM decides to call a tool

Instead of answering directly, the model emits a structured tool_use block:

{ "type": "tool_use", "id": "call_abc123", "name": "get_weather", "input": { "location": "Vienna, AT" } }

⚠️ The model produces JSON — it does NOT execute anything itself.

3
⚙️ Your code executes the tool & returns the result
🔧 Execution

Your orchestrator parses the tool_use block, calls the real external API, and gets the result.

📨 Feed result back

Append a tool_result message and call the LLM again:

{ "role": "user", "content": [{ "type": "tool_result", "tool_use_id": "call_abc123", "content": "18°C, partly cloudy" }] }

↻ Steps 2–3 may repeat if the model needs multiple tools

4
💬 LLM generates the final natural-language answer
🧠 LLM response

"It's currently 18 °C and partly cloudy in Vienna today — light jacket weather!"

👤 User

Sees a clean, natural answer — the tool plumbing is invisible.

💡 Key Takeaways

The LLM never executes tools. It only produces structured JSON describing which tool to call and what arguments to pass. Your application code (the "orchestrator") is responsible for actually running the function, then feeding the result back into the next API call.

It's a loop, not a single call. If the model needs data from multiple tools — or one tool's output informs the next — the orchestrator keeps looping (prompt → tool_use → execute → tool_result → prompt …) until the model returns a plain text response.

Tool schemas act like a menu. The model picks from the tools you provide based on their names, descriptions, and parameter schemas. Good descriptions = better tool selection.
